2. Critical Bug — r_pulley Discrepancy

3 곳에서 r_pulley 값이 일치하지 않음:

FileLineValueStatus
configs/walker_geometry.yaml130.035 mCorrect (사용자 measured)
firmware/teensy_host_receiver/force_clamp.h290.130 mWRONG (3.7x error)
firmware/teensy_host_receiver/README.md880.130 mWRONG (doc)
scripts/cable_kinematics.py(uses YAML)0.035 mCorrect

영향 분석

force_clamp.h 식: tau_motor_Nm = tau_cable_N × PULLEY_RADIUS_M

현재 (잘못 — 0.130m):
   사용자가 "5 N pretension" 명령
   → Teensy: 5N × 0.130m = 0.65 N·m motor torque
   → 실제 cable force = 0.65 N·m ÷ 0.035m_실제 = 18.6 N  ← 3.7x over

수정 후 (0.035m):
   5N × 0.035m = 0.175 N·m motor torque
   실제 cable force = 0.175 / 0.035 = 5 N  ✓ 의도와 일치

AK60 limit 도 잘못 계산:
   현재: AK60_MAX_TAU_NM = 9.0 (= 70N × 0.130 가정)
   현재 max cable force = 9.0 / 0.035 = 257 N  ← 사용자 안전 위험!
   수정: AK60_MAX_TAU_NM = 2.45 (= 70N × 0.035)
        max cable = 70 N ✓

수정 코드 (force_clamp.h)

inline constexpr float PULLEY_RADIUS_M      = 0.035f;   // motor pulley radius, walker_geometry.yaml 와 일치
inline constexpr float AK60_MAX_TAU_NM      = 2.45f;    // = 70N cable max × 0.035m
inline constexpr float AK60_MAX_TAU_CABLE_N = 70.0f;    // = AK60_MAX_TAU_NM / PULLEY_RADIUS_M
inline constexpr float AK60_MAX_SLEW_NM_S   = 7.0f;     // cable 200N/s 기준
inline constexpr float PRETENSION_TAU_NM    = 0.175f;   // = 5N × 0.035m (사용자 spec)

왜 이 bug 가 지금까지 안 잡혔나

  • Cable 미연결 운용 → motor torque 가 cable force 로 전달 안 됨
  • Walking session 의 Phase 3 (pipeline replay) 만 사용 → motor 명령 안 보냄
  • walker_geometry.yaml 의 0.035 는 vision/control 측 cable_length 계산용
  • force_clamp.h 의 0.130 는 hardware 측 torque conversion (분리되어 있어 감지 안 됨)

Sprint 3 (cable 연결 + bench test) 직전 까지는 *발견조차 안 되었을 가능성*. 지금 fix 가 critical.
