2. Title + Abstract

Title

Vision-Based Iterative Learning Control with Slack-Free Avoidance
for Cable-Driven Mobile Robotic Platforms

Abstract (RA-L)

Cable-driven mobile robotic platforms require real-time, slack-free actuator
control synchronized to user motion. Existing approaches rely on discrete event
triggers (e.g., IMU heel-strike) or fixed kinematic commands (e.g., 150° per
cycle), which lack continuous phase information, subject-specific accuracy, and
per-cycle learning capability. We present a vision-based control system that
integrates (1) continuous gait phase estimation via a 3-cascade EKF with
sub-20ms latency, (2) forward kinematics computing subject-specific cable
length reference from 30-second calibration, (3) a hybrid pre-emptive +
reactive slack-free controller with provable tension bounds, and (4) a
phase-indexed iterative learning controller that refines per-cycle feedforward
via P-type update with low-pass robustness filter. We prove asymptotic
convergence under bounded inter-cycle variability and validate on a
cable-driven mobile robotic walker across 5x assist magnitude (10-50N peak).
The system achieves cable length tracking RMSE under 1mm after 10 cycles of
ILC learning, slack event probability below 1%, and disturbance recovery
within 200ms. The architecture generalizes to cable-driven mobile robots
requiring user-cooperative actuator control with cyclic motion patterns.

5. 3-Tier Controller Architecture

┌─────────────────────────────────────────────────────────────────┐ │ Tier 0 — Reference Generation (Jetson, 60-120 Hz) │ │ ZED → YOLO TRT → Plan D EKF → φ, ω, q[6] │ │ Forward kinematics → P_shank → ℓ_ref(φ_fwd) │ │ ILC store: u_k(φ) lookup │ │ 50ms forecast: φ_fwd = φ + ω × 0.050 │ │ Output via SHM: ℓ_ref, F_FF, u_k │ └──────────────────────────────────│──────────────────────────────┘ ↓ ┌─────────────────────────────────────────────────────────────────┐ │ Tier 1 — Cooperative Admittance (Teensy, 1 kHz) │ │ Mv ẍ + Bv ẋ + Kv(φ) (x − ℓ_ref) = F_meas − F_FF │ │ Mv=0.1kg, Bv=2 Ns/m │ │ Kv(φ): stance 1 N/m, swing 50 N/m │ │ → ℓ_cmd │ └──────────────────────────────────│──────────────────────────────┘ ↓ ┌─────────────────────────────────────────────────────────────────┐ │ Tier 2 — Inner loops (Teensy, 4 kHz) │ │ τ_pos = Kp×e_pos + Kd×ė_pos (PD position) │ │ τ_F = Kp_F×e_F + Ki_F×∫e_F (PI force, load cell)│ │ τ_DOB = (c1 sign(ℓ̇) + c2 ℓ̇) × r_pulley (Bowden friction) │ │ Δτ_s = K_slack × max(0, F_min − F_meas) (reactive slack) │ │ u_k(φ) (ILC FF, phase-idx) │ │ τ_total = α τ_pos + (1-α) τ_F + τ_DOB + Δτ_s + u_k(φ) │ │ Safety: τ_motor = clip(τ_total, 0, 1.75 Nm) │ └──────────────────────────────────│──────────────────────────────┘ ↓ ┌─────────────────────────────────────────────────────────────────┐ │ Tier 3 — FOC Current Loop (AK60 펌웨어, ~10 kHz) │ │ I_des = τ_motor / K_t → PI on I → SVPWM → BLDC │ └──────────────────────────────────│──────────────────────────────┘ ↓ CABLE TENSION ON USER ↓ Load cell + Encoder → loop closure

6. ILC Formulation

Update law (P-type)

u_{k+1}(φ) = Q(z) × [u_k(φ) + L × e_k(φ)]

e_k(φ) = ℓ_ref(φ) − ℓ_actual_k(φ)
L      = 0.3 (initial), tunable 0.1-0.5
Q(z)   = low-pass, cutoff 8 Hz
         (gait fundamental 1-2 Hz × 5 harmonics margin)

Phase-indexed (not time-indexed)

  • 매 cycle 마다 φ ∈ [0, 2π) progression 다름
  • Resample e_k(t) → e_k(φ) using Plan D EKF φ(t)
  • Inter-cycle variability robust

Convergence (paper appendix)

Condition: |1 − L × P(z)| × |Q(z)| < 1 for all |z| = 1

Plant P(z): admittance + motor + cable dynamics, linearized

Asymptotic bound: ||e_∞(φ)|| ≤ ε_noise / (1 − Q(0))
                  → tunable to < 1mm under typical noise

7. Slack Control (KEY C3)

Pre-emptive layer (Tier 0)

φ_fwd = φ + ω × 50ms
ℓ_target = ℓ_ref(φ_fwd)
→ motor adjusts BEFORE slack 발생

Reactive layer (Tier 2)

if F_meas < F_min (0.5N):
    Δτ_slack = K_slack × (F_min − F_meas)
    τ_total += Δτ_slack
→ slack 발생 즉시 복구 within 1ms

Provable bound

Under bounded sensor noise ε_s and bounded user acceleration a_u:
    F_meas(t) ≥ F_min − ε_total
    ε_total = f(latency, K_slack, Bv, Mv, ε_s, a_u)
Tunable to ε_total < 0.2N → 99.9% slack-free
